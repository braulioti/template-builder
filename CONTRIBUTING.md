## Contributing

Contributions are always welcome! There are several ways to contribute to the Template Builder project:

### Contributing with Code

If you have programming knowledge, you can contribute directly with code:

1. **Fork the project** on GitHub
2. **Create a branch** for your feature or fix (`git checkout -b feature/MyNewFeature`)
3. **Make your changes** following the project standards
4. **Commit your changes** with clear and descriptive messages (`git commit -m 'Add new feature X'`)
5. **Push to the branch** (`git push origin feature/MyNewFeature`)
6. **Open a Pull Request** describing your changes

### Code Standards

When contributing code, please follow these guidelines:

- Follow the C++17 standard used in the project
- Use modern C++ practices (RAII, smart pointers, etc.)
- Write unit tests for new features using Google Test
- Ensure your code compiles on all supported platforms (Windows, Linux, macOS)
- Follow the existing code style and formatting
- Update documentation if adding new features

### Reporting Bugs

Found a bug? Help us improve the project by reporting it:

- Open an [Issue](https://github.com/braulioti/template-builder/issues) on GitHub
- Describe the problem clearly and objectively
- Include steps to reproduce the bug
- If possible, include error logs or stack traces
- Inform the Template Builder version and your operating system
- Include the YAML template file if the bug is related to template processing

### Suggesting New Features

Have an idea to improve Template Builder? Share it with us:

- Open an [Issue](https://github.com/braulioti/template-builder/issues) on GitHub
- Describe the proposed feature in detail
- Explain the problem it would solve or the benefit it would bring
- If you have examples or use cases, share them as well
- Consider how it fits with the existing YAML template format

### Documentation

Contributions to documentation are also welcome:

- Update README.md with new features or changes
- Add examples to the samples/ directory
- Update TROUBLESHOOTING.md with new solutions
- Improve code comments and documentation

All contributions, whether code, bug reports, suggestions, or documentation improvements, are valuable for the project's growth. Thank you for helping make Template Builder better!
