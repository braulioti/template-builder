Files

Files are the output of your template: each entry under the files key defines a file to be created in the generated project. You can supply the text inline or via a prompt; Template Builder creates each file and writes the result when the template is run.

Attributes

Name Required Description
path required Destination path of the file in the generated project. Relative to the project output directory (e.g. inc/customize-register.php, style.css).
content optional Inline content of the file. Can be plain text or use variable placeholders (e.g. {{ variablePlaceholder }}). If omitted and no prompt is set, the file is created empty.
prompt optional Name of a prompt defined in the prompts section. The prompt’s result becomes the file content. Useful when content is long or shared across multiple files.

Generate a File

For each entry in files, Template Builder creates the file at the destination path. If any subfolders in that path do not exist, they are created automatically. When both optional attributes are omitted—no inline content and no prompt—the file is generated empty. This is useful for placeholder files or when you intend to fill them later.

Prompt

During file generation, a prompt can be called to fill variables. When a file entry references a prompt, the prompt's inputs are shown to the user, their responses are bound to the associated variables, and the prompt's result becomes the file content. For more details on how prompts work, see the Prompts section.

Content

The content attribute holds the full content of the file. You can use variables and functions inside the content; the Template Builder replaces placeholders with the resolved values when generating the file. If content is omitted and no prompt is set, the Template Builder creates an empty file.
