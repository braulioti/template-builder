Prompts

Prompts are interactive questions shown to the user when running a template. Each prompt collects input through one or more inputs, binds those values to variables, and produces a result that can be used as file content. When a prompt input is bound to a variable, the variable's value is overwritten by the user's response. Prompts are executed in the order that files are created in the YAML. A variable modified by a prompt can be used in another file, and it can also be modified again by another prompt in a different file. Prompts make templates dynamic by allowing users to customize the generated output based on their responses.

Attributes

Name Required Description
name required Unique identifier for the prompt. Used when referencing the prompt in a file entry (e.g. prompt: promptName).
inputs optional Array of input questions shown to the user. Each input binds a user response to a variable. If omitted, the prompt only produces its result without collecting any input.
result optional Content that becomes the file content when the prompt is referenced in a file entry. Can use variable placeholders (e.g. {{ variablePlaceholder }}) that are replaced with the values collected from inputs. If omitted, the file will be created empty.

Inputs

Each input in a prompt's inputs array represents a question shown to the user. The user's response is bound to the specified variable and can be used in the prompt's result or in file content.

Input Attributes

Name Required Description
input required Text of the question displayed to the user (e.g. "Enter your project name: ").
variable required Name of the variable to which the user's response will be bound. The variable must be defined in the variables section.
type required Type of input control. Determines how the question is presented and how the user responds. See Input Types below.
options optional Array of options for Checklist type. Each option has name (displayed) and value (assigned to variable). Required for Checklist, ignored for InputString and ArrayList.

Input Types

Name Description
InputString Text input field. User types a free-form text response. The response is assigned directly to the bound variable.
Checklist Multiple choice selection. User can select one or more options from the provided options list. The selected option values are assigned to the variable.
ArrayList Free-form text input that accumulates multiple entries. User types text and presses Enter to add each entry. Entries are accumulated until the user enters an empty line. All entries are stored as multiple lines separated by line breaks and assigned to the variable.
