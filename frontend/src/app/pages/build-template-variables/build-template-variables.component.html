Variables

Variables are named placeholders that hold values used when generating files. Their values come from interactive prompts—each prompt input is bound to a variable—or from those defaults when no prompt is used. Once set, variables can be referenced in file content with {{ variableSyntax }}, and inside template functions like {{ upperSyntax }}. In short: variables are the link between what the user types in the CLI and what ends up in the generated files.

Attributes

Name Required Description
name required Unique identifier for the variable. Used when binding prompt inputs and when referencing the value in file content (e.g. {{ variableSyntax }}).
type required Data type of the variable.
value optional Default value for the variable. Used when no prompt supplies a value, or as the initial value before prompts run.

Types of variables

Name Description
string Text value. Used for names, labels, and any content that is displayed or written to files.

Using Variables

Variables can be used in the content of a file defined in the Files section. Put the variable name inside double curly braces (e.g. {{ variableSyntax }}); when generating the project, the Template Builder replaces each occurrence with the variable’s value.

Changing Variable Value

The value of a variable can be changed through a prompt. When a prompt input is bound to a variable, the user's response overwrites the variable's default value (if any). This allows templates to be interactive and customizable. For more information on how prompts work, see the Prompts section.
